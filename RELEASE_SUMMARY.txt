Date: 2025-01-06

--------------------------------------------------------------------------------
OVERVIEW
--------------------------------------------------------------------------------

Major update adding 12 new static analysis processors for Magento 2, bringing
total from 4 to 16 processors. Complete coverage of DI patterns, code quality,
templates, and architecture best practices.

--------------------------------------------------------------------------------
NEW PROCESSORS (12)
--------------------------------------------------------------------------------

DI & Architecture (2):
  • Preferences - Multiple preferences detection
  • ProxyForHeavyClasses - Heavy class proxy validation

Code Quality (5):
  • HardWrittenSQL - Raw SQL detection
  • SpecificClassInjection - Concrete class injection
  • UseOfRegistry - Deprecated Registry usage
  • UseOfObjectManager - ObjectManager usage
  • PaymentInterfaceUseAudit - Deprecated payment methods

Templates (3):
  • Cacheable - Non-cacheable blocks
  • AdvancedBlockVsViewModel - Template anti-patterns
  • Helpers - Deprecated helper patterns

Architecture (2):
  • BlockViewModelRatio - Block/ViewModel ratio analysis
  • UnusedModules - Disabled modules detection

--------------------------------------------------------------------------------
QUICK START
--------------------------------------------------------------------------------

Docker:
  docker pull ghcr.io/crealoz/easyaudit:latest
  docker run --rm -v $PWD:/workspace ghcr.io/crealoz/easyaudit:latest \
    scan --format=sarif --output=/workspace/report.sarif /workspace

PHAR:
  ./easyaudit.phar scan /path/to/magento --format=json

Build:
  box compile

--------------------------------------------------------------------------------
TESTING
--------------------------------------------------------------------------------

Run test suite:
  ./easyaudit.phar scan tests/fixtures/

Test specific processor:
  ./easyaudit.phar scan tests/fixtures/Helpers/
  ./easyaudit.phar scan tests/fixtures/ProxyForHeavyClasses/

--------------------------------------------------------------------------------
WHAT'S DETECTED
--------------------------------------------------------------------------------

Critical (Error Level):
  • Direct ObjectManager usage
  • Raw SQL queries
  • Same-module plugins
  • Multiple preferences conflicts
  • Missing proxies for heavy classes
  • Deprecated payment methods
  • Helper usage in templates

Important (Warning Level):
  • Framework class plugins
  • Block-heavy architecture
  • AbstractHelper extension

Suggestions (Note Level):
  • Non-cacheable blocks
  • Disabled modules in codebase
